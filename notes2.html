Command Line Interface (CLI) - e.x. Terminal, Git Bash, Powershell, etc.

  • pwd = print working directory, "where am I?", will print out path to current directory (what GUI calls a folder)
    When CLI is first opened it starts in the home directory (in my case this is C:/Users/bjerickson)
  • ls = list, helps us view the contents of a directory
  • start, use start . to view current directory's contents in a window, or use start directory_name to open another directory (folder)
  • cd = change directories, used to "move" somewhere else, cd Desktop will move to Desktop directory (folder) if it's within current directory, cd .. will move up one directory, cd will return to home directory, cd FolderA/FolderAb/FolderAbC can accomplish several steps at once (this is referred to as "chaining" the directories together)
  • mkdir = make directory ("make-dur" or "make-deer"), creates a new folder with the name we provide: mk dir animals will make new folder called animals
    to make folder "rodents" inside of "animals":
    cd into animals than mkdir rodents
    OR from Desktop, "mkdir animals/rodents"

    to make several new directories, separate them with a space: mkdir cats dogs marsupials
    for this reason, it is not a good idea to put spaces in your directory or filenames!!

  • touch<, creates new files in one step, ex touch first_file.txt will create brand new, empty text file, you can verify that it was made using ls, can make a file in a directory other than current directory by specifying filepath (as with lsls and cd), and to create multiple files separate with spaces (as with mkdir)
  • rm is used to delete a file completely (not move it to the trash), can delete multiple files by separating names with spaces as before
  • There are two ways to remove directories:
    rmdir, rmdir directory_name deletes a directory, but only works if it's empty
    rm -rf (be careful with this one!), deletes a non-empty directory where -rf = recursive force, this is potentially very dangerous!!!!!!!

Grammar (more notes on this section can be found here)
English: sentence = subject, verb, object
subject and object can be nouns, etc
Languages have rules and parts of speech, etc

Backus Naur Form
precisely describe language in simple and concise way
Non-terminal --> replacement
replacement can be non-terminal or terminal and you keep replacing until you reach terminals:
Sentence --> Subject Verb Object
Subject --> Noun
Object --> Noun
Verb --> Eat
Verb --> Like
Noun --> I
Noun --> Python
Noun --> Cookies

Derivation starts from non-terminal and follows rules until we have only terminals left
Valid outputs for above include: "I Eat Cookies" and "Python Like Python", etc.

Python grammar is much stricter than most natural language grammars
Code must match language grammar exactly (see here for an example of bad Python grammar)

Python Grammar for Arithmetic Expression:
Expression --> Expression Operator Expression (recursive definition, resulting in an infinite number of possible outcomes/derivations)
Expression --> Number
Operator --> +
Operator --> *
Number --> 0, 1, ... etc
Expression --> (Expression)
An example derivation can result in: 1 + 1 and/or replacing expressions with expressions can increase complexity infinitely!
A few simple rules (like above) can be used to describe a big language!

Indexing Strings (more here)
String indexing lets you select individual characters in the string or extract subsequences
string[expression] e.x. 'udacity'[0]expression can be anything that evaluates to a number, e.x. 1 + 1
more common to use variable, not a string:

name = 'Dave''

will return v

Selecting Subsequences
string[expression1:expression2]
will evaluate to a string that is a subset of the original string starting from index expression1 and ending at index expression2 (non-inclusive, i.e characters expression1 through expression2 - 1)
leaving either side of the : blank will start at either the beginning [:4] or go through the end [3:], this way you don't actually need to know how long the string is!
s[0:0] will give an empty string
You can index positions that don't exist in this way!

Finding Strings in Strings (more here)
'find' method, built in procedure provided by Python, operates on strings:
string.find(string)
output is position where that substring is found (first occurrence) as a number, i.e. where in target string the search string is found
If target string is not found, the output will be -1
Note that Python is case sensitive!
According to Python, every string begins with an empty string (?)
'parameters' are what go inside parentheses

To find other occurrences, pass a second parameter:

my_string = "hello, hello"
print my_string.find('h')
print my_string.find('h', 1)
print my_string.find('e')
print my_string.find('e', 3)

Should return 0 for the first occurrence of 'h'
the second should return 7
third should return 1
last should return 8 because it started looking for 'e' at the second l

Note that 1 tells find what position to begin search, in the above string 1 refers to the first e, not the first occurence of 'h'

Replacing Strings


The method replace() returns a copy of the string in which the occurrences of old have been replaced with new, optionally restricting the number of replacements to max
Like this: str.replace(old, new[, max])
More simply: test_string.replace(old, new)

An example:

sentence = "I am a NOUN"
print sentence.replace("NOUN", "bird")

Replace method allows us achieve the same outcome as slice and concatenate, but much more simply!
